# Veloci UI Design Spec

**Date:** 2026-07-02  
**Status:** Approved  
**Component library:** shadcn/ui (general UI) + lightweight-charts (Horizon graph)

---

## Overview

Veloci is a local-first personal finance app built around $/day rates. The UI has one fixed layout: a sidebar for navigation and accounts, a canvas for the active view. There are no modal overlays for core data — everything is visible on one screen at a time.

The atomic unit is the **rate** ($/day, $/mo, or $/yr). All values display in the user's chosen rate format. Budget health is communicated through data, never through alarm banners or editorial language.

## Sidebar

### Navigation
Three primary nav items: **Budget**, **Reports**, **Review**. Active item has a filled background. Review shows a badge count for items pending categorization.

### Accounts
Two groups: **Active** (checking, savings, credit cards) and **Passive** (loans, retirement, investment accounts not on budget). Each row shows account name and current balance. Passive liabilities show negative balances in a muted red. A `+` affordance in each group header opens account import/connection — this is an account-level concern, not a global one.

### Footer
**Settings** and **Glossary** live at the bottom of the sidebar. Glossary is always accessible without leaving the current view context.

## Budget Canvas

### Topbar

Contains the page title ("Budget") and the **Rate format toggle** only:

```text
Budget                                    /day  /mo  /yr
```

No window-size picker. Rate format drives both the display unit and the Horizon viewport size (see graph section). No scrubber control — timeline navigation is handled by dragging the graph directly.

---

### Summary Strip

Equation layout: **Income − Commitments = Margin**

```text
Income          −   Commitments     =   Margin
$8,625              $6,190              $2,435
/mo · Jun 2026      /mo · Jun 2026      /mo · 28%
[actual]            [actual]            [actual]
──────────────────────────────────────────────────
[████████████████████████░░░░░░░░░]  proportion bar
 72% commitments                    28% margin
```

**Time-relative behavior:** The badge on each cell reflects the selected Pulse period:
- Pulse ≤ TODAY → badge reads `actual`, values come from transaction data
- Pulse > TODAY → badge reads `projected`, values come from amortized projection, cells have a dashed border treatment

**Negative margin:** The Margin cell background turns dark red, text turns red. No alarm banner — color is the only signal. Margin is negative when commitments exceed income.

**Proportion bar:** 3px bar below the strip. Commitment width proportional to their share of income. Margin fills the remainder. If margin is negative, bar is 100% red.

All values in the strip match the active rate format (/day, /mo, or /yr).

### Horizon Graph

The Horizon graph is a candlestick chart showing **margin rate** over time. It is built with **lightweight-charts** (TradingView open-source).

#### Viewport and rate format

Rate format drives the fixed viewport size and candle period:

| Format | Candle period | Viewport | Candles shown |
| --- | --- | --- | --- |
| /day   | 1 day        | 14 days  | 14            |
| /mo    | 1 month      | 1 year   | 12            |
| /yr    | 1 year       | 1 decade | 10            |

The viewport is fixed. To move through history or into the projected future, the user drags the graph horizontally. No scrubber control.

#### Projection line

A smooth dashed curve representing the expected margin rate at each point in time, calculated from all known commitments amortized forward. The curve is always smooth — amortization absorbs known future events (e.g. a loan payoff) gradually, so there are no discrete steps. The projection continues past TODAY into the future based on current known commitments.

#### Candle anatomy

Each candle represents one period (day/month/year):

- **Body:** The gap between projected margin and actual close margin for the period. Body height = drift magnitude.
- **Wick:** Min/max actual margin recorded within the period.
- **Color:**
  - Green — actual margin closed *above* projection (ahead)
  - Red — actual margin closed *below* projection (behind)
- **Open reference:** The projection value for the period (the expected baseline).

#### Pulse

The Pulse is the selected candle. Clicking any candle sets it as the Pulse. The Stack and Summary strip update to reflect that period. By default the Pulse is the most recently closed period (TODAY or the last complete candle).

When the Pulse is in the future (a projection candle), the Stack shows projected rates and the Summary shows `projected` badges.

#### Y axis range

The Y axis is scoped tightly to the actual drift range, not the full margin range. Typical drift is small (±$50–150/mo), which makes candles tall and readable. Major outliers (Hits, Boosts) appear as visually dominant candles without needing labels.

No "Boost" or "Hit" labels on the graph. The candle height communicates outliers. Labels appear only in the Stack.

#### Legend

```text
■ Ahead    ■ Behind    --- Projection    body = drift magnitude · wick = period range · drag to scroll
```

All legend terms have tooltip indicators (dashed underline, short hover definition).

#### TODAY marker

A subtle vertical dashed line at the current date divides history from projection. Candles to the right are projection-only and rendered slightly lighter.

### Stack Panel

The Stack is a waterfall breakdown of what went into the margin calculation for the selected Pulse period. It is collapsible (toggle in the panel header).

#### Header

```go
Stack  —  Jun 2026  [actual]                entry · type · actual /mo · drift /mo   ∧
```

The `actual` / `projected` badge mirrors the Summary strip, matching the Pulse position.

#### Columns

| Column | Width | Description |
| --- | --- | --- |
| Entry name + type | 148px | Name, commitment type (Standing / Variable / Annual) |
| Track bar | flex | Visual proportion of income consumed |
| Actual rate | 68px | Right-aligned, in active rate format |
| Drift | 64px | Signed rate delta, margin perspective |

**Drift sign convention (margin perspective):**
- `+$X` — this row ran *ahead* of projection (margin improved). Displayed green.
- `−$X` — this row ran *behind* projection (margin hurt). Displayed red.
- `$0` — on projection exactly. Displayed in muted text.

Standing commitments with no variance always show `$0`. Variable entries show drift when actuals differ from the amortized projection.

#### Row structure

**Income anchor row** (top): Full green track. Drift column empty (—).

**Category headers**: Collapsed by default at higher rate formats, expanded at /day. Each group uses a chevron expand/collapse.

**Commitment rows**: Each shows its actual rate for the period. Track bar fills from left (remaining margin after this row is removed).

**Hit / Boost tagging:**
- Rows where the actual exceeded the projection are tagged `Hit` with a red pill badge.
- Rows where the actual was below the projection (or income exceeded) are tagged `Boost` with a green pill badge.
- Tagged rows receive a highlighted background (red-tint or green-tint) and a 2px colored left border.
- Hit/Boost tags have tooltip indicators explaining the term.

**Margin anchor row** (bottom): Shows the remaining margin after all commitments. Track bar shows the remaining green slice. Drift shows the total period drift.

## Glossary Page

Accessible from the sidebar footer. A full-page reference for all Veloci domain vocabulary. Each entry has the term, a full definition, and examples where relevant. Tooltips in the app contain only the short definition — examples live here.

### Terms defined

| Term | Short definition (tooltip) |
| --- | --- |
| Margin | Income minus all commitments at the selected rate. |
| Projection | Expected margin, calculated from known commitments amortized over time. |
| Pulse | The selected period on the Horizon graph. Stack and Summary reflect this period. |
| Horizon | The candlestick graph showing margin over time, past and future. |
| Drift | Difference between actual and projected rate. +$X = ahead, −$X = behind. |
| Hit | An unexpected cost that pushed a commitment above its projected rate. |
| Boost | An unexpected positive — a commitment cost less, or income exceeded expectation. |
| Stack | Waterfall breakdown of the Pulse period showing each commitment's rate and drift. |
| Amortization | How Veloci spreads known future events (e.g. annual insurance) evenly over time as a stable rate. |

## Tooltip Convention

All domain-specific terms in the UI receive:
- A **dashed underline** (CSS: `border-bottom: 1px dashed`) to indicate interactivity
- A **hover tooltip** (shadcn/ui `<Tooltip>`) with a one-sentence definition only
- No examples in tooltips — examples live in the Glossary

Tooltip trigger terms: Margin, Income, Commitments, Projection, Pulse, Horizon, Drift, Hit, Boost, Stack, Amortization, Ahead, Behind.

## Data Model Assumptions (UI layer)

The UI consumes pre-computed data from the API. It does not perform financial calculations.

Each Pulse period the API returns:
- `income_rate` — actual or projected, in the requested unit
- `commitments_rate` — actual or projected
- `margin_rate` — derived: income − commitments
- `projection_rate` — smooth amortized expected margin
- `drift_rate` — margin_rate − projection_rate (signed, margin perspective)
- `candle` — `{ open: projection_rate, close: margin_rate, high, low }` per period
- Per commitment: `{ name, type, actual_rate, projected_rate, drift_rate, tag: 'hit' | 'boost' | null }`

The API determines Hit/Boost classification. The UI renders the tag without applying its own threshold logic.
